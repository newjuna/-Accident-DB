v15 안전보건팀 산재승인 조회 + 요약 4페이지 추가본

기준 파일
- v14_safety_team_integrated_approval_sync_clean_menu_jkl.zip

이번 수정 핵심
1. 안전보건팀 로그인 전용 메뉴 추가
   - 위치: 반복사고 매장과 로그아웃 사이
   - 메뉴명: 산재승인 조회
   - 수도권영업부문/지방영업부문 로그인에서는 보이지 않음

2. 산재승인 조회 화면 추가
   - 원본DB AC~AE 기준
   - AC: 산재승인 유무
   - AD: 근로손실일수
   - AE: KPI집계현황분류

3. 산재승인 조회 화면 구성
   - 상단 KPI 카드 4개
     · 산재승인 건수
     · 총 근로손실일수
     · KPI 집계대상
     · 출퇴근 제외
   - 필터
     · 연도, 월, 영업부, 팀, 산재승인 유무, KPI분류, 매장명 검색
   - 표
     · 재해일자, 영업부, 팀, 매장명, 재해유형, 산재승인, 근로손실일수, KPI분류, 사고내용

4. 요약 생성 기능 확장
   - 기존 수도권/지방/전체 통합 3페이지는 유지
   - 안전보건팀 로그인 시 4페이지 추가
   - 4페이지: 산재승인 및 KPI 집계 현황
   - 다운로드 ZIP 파일명도 안전보건팀_4장으로 변경

5. 개인정보 보호
   - 주민등록번호는 화면/요약 어디에도 표시하지 않음
   - 산재승인DB의 주민등록번호는 대시보드에서 사용하지 않음

적용 파일
- Code.gs
- index.html
- app.js
- style.css
- logo.png

적용 순서
1. Apps Script에 Code.gs 덮어쓰기
2. Apps Script 저장 및 새 버전 배포
3. GitHub Pages 저장소에 index.html / app.js / style.css / logo.png 덮어쓰기
4. PC에서 Ctrl+F5 새로고침
5. 안전보건팀 로그인 후 산재승인 조회 메뉴 확인
6. 기준연도/기준월 선택 후 요약 생성 실행
7. ZIP 안에 4장 PNG가 생성되는지 확인

테스트 포인트
- 수도권/지방 로그인에서 산재승인 조회 메뉴가 안 보여야 함
- 안전보건팀 로그인에서만 산재승인 조회 메뉴가 보여야 함
- 산재승인 조회 상단 카드가 AC~AE 기준으로 집계되어야 함
- 요약 생성 시 기존 1~3페이지는 유지되고 4페이지가 추가되어야 함
