v32_loading_checklist_ui

변경 내용
1. 로딩 화면을 체크리스트형 진행 화면으로 변경
   - 기존: 진행률 %, 게이지 바, 현재 단계 문구만 표시
   - 변경: 작업 단계가 체크리스트처럼 표시되고, 진행 상황에 따라 순서대로 체크 표시
   - 예: 산업재해 원본 데이터 조회 → 산재 승인 사고 데이터 조회 → 영업부문/팀/매장 정보 정리 → 화면 구성

2. 로딩 문구 개선
   - 로그인, 초기 데이터 로딩, 대시보드, 데이터조회, 중상해 매장, 상세보기, 요약 생성, 이미지/ZIP 생성 상황별로 문구를 다르게 표시
   - 사용자가 기다리는 이유를 알 수 있도록 “사고 데이터를 수집하기 위해 시간이 필요합니다.” 형태의 안내 문구 적용

3. v31 기능 유지
   - 로그인 후 전체 데이터를 1회 로딩하고, 이후 월 변경/필터 변경/메뉴 이동은 웹앱 내부 캐시를 우선 사용
   - 사고경위서 사고 / 산재 승인 사고 / 중상해 매장 / 상세보기 로딩 최소화 구조 유지
   - 요약 생성과 이미지/ZIP 다운로드는 파일 생성 작업이므로 로딩창 유지

4. v28/v30 기존 기능 유지
   - Teams/카카오톡 썸네일 설정 유지
   - 사고 원본DB 안전보건팀 전용 노출 유지
   - 산재 승인 사고 엑셀 조건부 색상 유지
   - 출퇴근재해 제외 기준 유지

배포 방법
1. ZIP 압축 해제
2. GitHub Pages 저장소에 index.html / app.js / style.css / logo.png / thumbnail 관련 파일 전체 덮어쓰기
3. 이번 v32는 로딩 화면 UI 수정이 핵심이므로 Code.gs는 v31과 동일 구조입니다.
4. 이미 v31 Code.gs를 적용했다면 Apps Script 교체는 필수는 아닙니다.
5. 다만 버전 혼동 방지를 위해 ZIP 안의 Code.gs까지 함께 교체해도 됩니다.
6. GitHub Pages 주소에서 강력 새로고침 후 테스트

중요
- 월 변경/필터 변경 시 로딩을 최소화하는 핵심 구조는 v31에서 유지됩니다.
- 로그인 후 원본DB가 바뀌어도 현재 화면에는 즉시 반영되지 않습니다. 최신 데이터를 보려면 새로고침 또는 재로그인을 해야 합니다.
- 체크리스트는 실제 서버 내부 진행률을 100% 실시간으로 받아오는 구조가 아니라, 사용자가 기다리는 동안 현재 작업 단계를 이해하기 쉽게 보여주는 화면 연출입니다.

테스트 순서
1. 안전보건팀 로그인
2. 초기 로딩 화면에서 체크리스트 문구가 순서대로 표시되는지 확인
3. 로그인 후 대시보드 연도/월을 여러 번 변경해 로딩이 최소화되는지 확인
4. 사고경위서 사고 > 데이터 조회에서 월/영업부/팀/매장 검색 변경
5. 산재 승인 사고 > 대시보드/데이터조회/중상해 매장 메뉴 이동
6. 요약 생성 시 체크리스트형 로딩 화면이 정상 표시되는지 확인
