v14 안전보건팀 통합 + 산재승인DB 자동반영 기능 추가본

이번 반영 내용
1. 구글시트에 산재승인DB 시트를 추가해서 근로복지공단 승인자 파일을 붙여넣는 구조 추가
2. 원본DB의 AC~AE열 자동 입력 기능 추가
   - AC: 산재승인 유무
   - AD: 근로손실일수
   - AE: KPI집계현황분류

원본DB 자동 입력 기준
1. 산재승인 유무
   - 원본DB의 재해자명 + 재해일자
   - 산재승인DB의 이름 + 재해일자
   - 두 값이 일치하면 AC열에 Y 입력

2. 근로손실일수
   - 산재승인DB의 요양시작일 ~ 요양종료일 기준
   - 계산식: 요양종료일 - 요양시작일 + 1일

3. KPI집계현황분류
   - 산재승인DB와 매칭된 건만 분류
   - 재해발생형태가 출퇴근이면 KPI 제외(출퇴근)
   - 원본DB 재해유형이 넘어짐 / 무리한 동작 / 물체에 맞음이면 해당 분류 입력
   - 그 외 승인건은 KPI 제외
   - 미승인건은 빈칸

사용 방법
1. 구글시트에 산재승인DB 시트를 만든다.
2. 근로복지공단 다운로드 파일 내용을 산재승인DB에 그대로 붙여넣는다.
3. 구글시트 새로고침
4. 상단 메뉴 [산재승인 관리] → [원본DB AC~AE 자동 업데이트] 클릭
5. 원본DB AC~AE열 자동 입력 확인

산재승인DB 필수 헤더
- 이름
- 주민등록번호
- 재해일자
- 요양시작일
- 요양종료일
- 의료기관
- 재해발생형태
- 정밀진단
- 요양관리지사

주의
- 주민등록번호는 매칭에 사용하지 않음
- 이름 + 재해일자 기준으로 매칭
- 기존 원본DB 데이터는 삭제하지 않음
- AC/AD/AE 헤더가 비어 있으면 자동 생성
- AC/AD/AE에 다른 헤더가 있으면 덮어쓰지 않고 마지막 열 뒤에 생성
- Code.gs 변경 있음
